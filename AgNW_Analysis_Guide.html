02Quick start

Process a single SEM image

$ python nanowire_analysis.py F_1000x.tif --output_dir results/

Process a whole folder

$ python nanowire_analysis.py --input_dir ./images/ --output_dir results/

Mix SEM and VLM in one run

$ python nanowire_analysis.py *.tif *.png --output_dir results/

The script auto-detects image type by extension: .tif/.tiff → SEM pipeline, .png/.jpg → VLM pipeline. Results appear in the output directory immediately after processing.

Important — SEM images
SEM .tif files must contain the standard instrument metadata bar at the bottom (with scale bar and text annotation). The script reads the scale bar automatically. If the scale bar is missing or non-standard, set --fallback_nm_per_px manually.

03Supported inputs

ModeFormatMagnificationWhat is measured

Coverage .png 20× (sub-pixel wires) Network fill %, orientation anisotropy

VLM filename convention

The VLM scale calibration is read from the filename. Include the magnification as 50x or 100x anywhere in the filename:

# Good — magnification detected automatically
50x_SampleC_run2.png
100x_E_2-2_clean.png
my_sample_100x_second_pass.png

# Fallback — set scale manually if filename has no mag
$ python nanowire_analysis.py image.png --vlm_scale_um 10.0

20× images are detected by their calibrated nm/px value exceeding 300 nm/px and are routed to the coverage mode automatically.

04Output files

Each image produces several files in the output directory. All filenames are prefixed with the input image stem.

{name}_all_wires.csv

Per-wire measurements. One row per qualified wire. The primary data file for analysis.

{name}_overlay.png

Colour-coded overlay on the original image. Wire colour = length (plasma colourmap: blue → yellow). Only a random sample of up to 300 wires is drawn for clarity.

{name}_distributions.png

Distribution plots: length histogram + KDE, diameter (SEM only), aspect ratio (SEM only), and orientation rose plot.

{name}_raw_skeleton.png

Debug image: skeleton pixels in grey, endpoints in green, junctions in red. Useful for diagnosing segmentation issues.

all_images_summary.csv

Concatenated per-wire table for all images in a single run. Convenient for cross-sample statistics.

{name}_coverage.png

Coverage mode only (20× VLM). Binary network mask with fill percentage annotated.

Only complete wires count
Wires whose skeleton coordinates touch the 0.5 µm border zone are excluded from all statistics and labelled truncated=True in the CSV. The printed counts and distribution plots use only complete wires.

05How it works

1
Scale bar detection
Finds the SEM annotation bar (dark band at the bottom of the image) and locates the white scale-bar line within it to compute nm/pixel. VLM images use a lookup table calibrated per magnification.
2
Frangi vesselness preprocessing
A multi-scale Frangi filter (σ = 2–4 px) enhances tubular structures. This produces a much cleaner binary than simple thresholding, especially at wire crossings where contrast is low.
3
Skeletonisation & junction labelling
Zhang skeletonisation reduces wires to single-pixel paths. Each pixel is labelled by its Nano1D neighbour count: N=1 tail, N=2 section, N≥3 junction. Gap bridging reconnects small Frangi breaks.
4
Nano1D recall-vector tracing
Starting from every tail pixel, the tracer follows the skeleton pixel-by-pixel. At each junction it computes a recall vector from the last 30 pixels of travel and selects the arm most aligned with that direction. This handles wire crossings far better than segment-level methods.
5
Post-tracing merge pass
The tracer sometimes picks the wrong arm at ambiguous junctions, splitting one wire into two fragments. The merge pass finds fragment endpoint pairs where both endpoints are within 300 nm of a junction pixel (confirming they were terminated by a junction, not a natural wire tip), checks direction alignment, and greedily reassembles them.
6
Quality filters & border exclusion
Wires are rejected if: chord-deviation ratio > 0.40 (V-kinks), sinuosity > 1.50 (near-loops), length < minimum threshold, or any skeleton coordinate falls inside the 0.5 µm border zone.
7
Diameter measurement (SEM only)
At 12 evenly-spaced points along each wire, a greyscale profile is taken perpendicular to the local tangent. The FWHM of the intensity peak gives the apparent wire diameter. Reported values are not PSF-deconvolved.

06Key parameters

All parameters have sensible defaults for 1000× SEM and 50/100× VLM images at 25 nm/px. Only change them if you're seeing specific problems.

Minimum length thresholds

FlagDefaultEffect
--min_len_um5.0 µmSEM: discard wires shorter than this
--vlm_min_len_um10.0 µmVLM: discard wires shorter than this

Shape quality filters

FlagDefaultWhat it controls
--max_chord_ratio0.40 Rejects V-kinks. Ratio = maximum perpendicular deviation from the endpoint chord ÷ chord length. Lower = stricter (try 0.25 for very straight wire samples).
--max_sinuosity1.50 Rejects near-loop traces. Sinuosity = arc length ÷ chord length. Physical S-curves typically have sinuosity < 1.3.

Merge pass

FlagDefaultWhat it controls
--no_mergeoffDisable the merge pass entirely (fastest, fewest wires)
--merge_gap_um1.0 µmMaximum endpoint separation for merge candidate pairs
--merge_angle30°Proximity cone: other endpoint must lie within this angle of the wire's heading
--merge_join_angle25°Maximum bending angle at the join point — prevents back-bending merges
--merge_chord0.25Chord filter applied to the merged wire (stricter than individual wire filter)

Border exclusion

FlagDefaultWhat it controls
--border_um0.5 µmWidth of the exclusion zone around the image frame. Wires touching this zone are removed.

Diameter range (SEM only)

FlagDefaultNotes
--min_diam_nm10 nmWires with measured diameter below this are excluded
--max_diam_nm400 nmWires with measured diameter above this are excluded (catches artefacts)

Tracing internals (rarely need changing)

FlagDefaultNotes
--search_distance30 pxRecall vector length. Increase for very straight wires; decrease for very curved samples.
--no_frangioffFall back to CLAHE+Otsu binary (faster but worse at crossings)
--seed42Random seed for reproducible overlay sampling

07Common recipes

Run all SEM images in a folder, save to timestamped results

$ python nanowire_analysis.py --input_dir ./sem_images/ \
    --output_dir results_$(date +%Y%m%d)/

Tighter kink rejection for very straight wire samples

$ python nanowire_analysis.py sample.tif \
    --max_chord_ratio 0.20 --merge_chord 0.15

Shorter wires (e.g. high-concentration sample)

$ python nanowire_analysis.py sample.tif --min_len_um 2.0

Disable merge pass for speed or comparison

$ python nanowire_analysis.py sample.tif --no_merge

Override VLM scale when filename has no magnification

$ python nanowire_analysis.py mystery_image.png --vlm_scale_um 20.0
# --vlm_scale_um is the physical length of the scale bar in µm

Post-filter CSV for very straight wires only

# In Python / pandas, after loading the CSV:
df = pd.read_csv("F_1000x_all_wires.csv")
clean = df[(df["sinuosity"] < 1.2) & (df["truncated"] == False)]

09Troubleshooting

Too few wires detected

Check the raw skeleton image (_raw_skeleton.png). If skeleton coverage looks sparse compared to what you can see in the original:

  • Lower the Frangi threshold: --frangi_thresh 0.005
  • Lower the minimum length: --min_len_um 3.0
  • Check the scale bar is being read correctly — see the "bar at row …" line in the output

Too many V-kink artefacts

Lower the chord filter. This trades some valid curved wires for fewer kinks:

$ python nanowire_analysis.py sample.tif --max_chord_ratio 0.20

Wires joining across too large a gap

Tighten the merge gap:

$ python nanowire_analysis.py sample.tif --merge_gap_um 0.5

Edge wires appearing along the image frame

Increase the border exclusion zone:

$ python nanowire_analysis.py sample.tif --border_um 1.0

Scale bar not found (RuntimeError)

Set the fallback scale manually using your microscope's recorded nm/px:

$ python nanowire_analysis.py sample.tif --fallback_nm_per_px 30.0

VLM calibration warning

If the script prints [CORRECTED — calibrated from filename mag], the auto-detected scale bar length disagreed with the filename magnification by more than 20%. The filename-based calibration was used. Verify that the image filename contains the correct magnification (e.g. 50x or 100x).

Diameter values are apparent widths
The FWHM diameter measurement includes the electron-beam point-spread function and is systematically larger than the true wire diameter. Do not compare raw diameter values between instruments without PSF deconvolution. Use the values for within-instrument comparisons and for aspect ratio calculations only.

10Physical notes

What "tip-to-tip" means

A wire is classed as tip-to-tip (ep_to_ep = True) when both of its endpoints in the skeleton are genuine N=1 tail pixels — i.e. pixels with exactly one neighbour. This means the tracer started and ended at a natural wire tip rather than at a frame edge or an unresolved junction. High tip-to-tip rates (≥ 90%) indicate good segmentation and tracing quality.

Orientation convention

Wire angles are reported as the acute angle (0–180°) of the wire's principal axis measured from the vertical (column direction). An angle of 90° means a horizontal wire. The orientation rose plot uses 18 bins of 10°. The mean resultant length R quantifies anisotropy: R = 0 is fully isotropic; R = 1 is perfectly aligned. Rayleigh and χ² p-values test the null hypothesis of uniform distribution.

Coverage mode (20× VLM)

At 20× magnification individual wires are sub-pixel (~366 nm/px) and cannot be traced individually. The pipeline switches automatically to a binary segmentation mode that reports the percentage of the field of view covered by the wire network, together with an orientation estimate from segment PCA on the binary skeleton.

Why sinuosity matters

Sinuosity (arc ÷ chord) measures how much a wire deviates from a straight line. Physically real AgNWs are stiff and have sinuosities close to 1.0–1.15 even when naturally curved. Values above 1.3 often indicate a tracing artefact (the recall vector picked a wrong arm and the trace meandered). The default filter of 1.5 removes clear loops while keeping genuinely curved wires.
